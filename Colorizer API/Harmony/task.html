Description

Some colors may look good when used together. It is called color harmony. The color wheel can help you with these harmonies. You may also have heard the term complementary harmony. Green works well with red, blue with orange, etc.
At this point, you need to handle a monochromatic harmony. It doesn’t employ the color wheel because a monochromatic harmony doesn’t change the hue, so it’s the simplest type of harmony to calculate.

In this stage, create a harmony view that takes an HSV color in the request and if the “harmony” is “monochromatic”, it should return three colors: one base color, one shade, and one tint. Please, use color-harmony/ path_info to map hostname:port/color-harmony/ to the harmony view.

A color shade is achieved by adding black to the color; decrease V (brightness) in HSV.
A color tint is achieved by adding white to the color; increase V (brightness) in HSV.

The difference between HSV shade and tint values from the base color is 20. If the base color value (V) is 10 then the result would contain the following values: 10, 30 (shade), and 50 (tint).

Objectives

  1. Implement the monochromatic color harmony by adding a new harmony view.
  2. Use the customized Serializer class to validate the color values.

Please, round your numbers at the very last stage only.

Examples

The greater-than symbol followed by a space (> ) represents the user input. Note that it's not part of the input.

Example 1: A correct request

Request:

{
  "representation": "hsv",
  "color": [
    128,
    85,
    95
  ],
  "harmony": "monochromatic"
}

Response:

  • HTTP status code: 200
{
  "representation": "hsv",
  "color_1": [
    128.0,
    85.0,
    55.0
  ],
  "color_2": [
    128.0,
    85.0,
    75.0
  ],
  "color_3": [
    128.0,
    85.0,
    95.0
  ]
}

Example 2: A wrong request:

Request:

{
  "representation": "hsv",
  "color": [
    128,

  ],
  "harmony": "mono"
}
