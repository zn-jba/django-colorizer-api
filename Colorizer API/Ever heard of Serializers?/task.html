Description

Your API validates colors in each view independently. Repeating code is not good, so keep your code as simple as possible. You can do it with Serializers from the REST framework.
Use the rest_framework.serializers.Serializer class; it handles incoming data in the request. After calling the is_valid() method, it validates the data and returns the validated_data and the data. You can use these values to check and correct the color values. Modify your views to use the serializer class and check whether the incoming values in the request are correct.

Convert your views between RGB and HSV in addition to HEX and HSL.

  • RGB to Hex
  • RGB to HSL
  • RGB to HSV
  • Hex to RGB
  • HSL to RGB
  • HSV to RGB

Regexps can help you to validate the Hex values

Objectives

  1. Use the serializers.Serializer to validate data in the request.
  2. Any invalid value in the incoming JSON object should return the following message in the response: 
    {
  "error": "Invalid data."
}
  3. Add HEX and HSL conversion features. Your app will have to convert between all four representations: HSV, HSL, HEX, and RGB.

Examples

The greater-than symbol followed by a space (> ) represents the user input. Note that it's not part of the input.

Example 1: A correct request

Request:

{
  "representation": "hsv",
  "color": [
    128,
    85,
    95
  ],
  "conversion": "hex"
}

Response:

  • HTTP status code: 200
{
  "color": [
    128.0,
    85.0,
    95.0
  ],
  "converted_color": "#24F240"
}

Example 2: A wrong request

Request:

{
  "representation": "hex",
  "color": "#24ZZZZ",
  "conversion": "hsv"
}

Response:

  • HTTP status code: 400
{
  "error": "Invalid data."
}
