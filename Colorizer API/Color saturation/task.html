Description

Color can be described by many models, RGB being the most common. Other color models are HSV, HEX, and HSL. HSV is straightforward and very intuitive. H stands for hue (a value between 0 and 360). S stands for saturation (a value between 0 and 100). V stands for value/brightness (a value between 0 and 100). A change in any of these values provides predictable results across the light spectrum.

Django Framework and Django Rest Framework (DRF) can produce a view that takes a POST request with data in JSON format. JSON data should have an HSV color representation with two instructions â€” an operation to perform (saturation or desaturation) and the amount as a percentage, so if the amount is 5 and saturation is 85, then the color we get is \(85 + 85 *0.05 = 89.25\). The response is a JSON object that contains the modified color. Please use the modify-color path_info to map hostname:port/modify-color/.

You can use either Postman or BrowsableAPIRenderer. They are provided by default by DRF for tests and experiments. The view can be accessed by navigating to localhost:8000/modify-color/ URL using your preferred web browser.

Objectives

  1. Create a Django view that receives a request with color and returns a modified version of it in the response.
  2. The saturated color should have rounded values, for example. if saturation is 80.75, round it to 81.
  3. Use the modify-color/ path_info for URL mapping.

Example

The greater-than symbol followed by a space (> ) represents the user input. Note that it's not part of the input.

Example 1: A desaturation request

Request:

{
  "representation": "hsv",
  "color": [
    128,
    85,
    95
  ],
  "operation": "desaturate",
  "amount": 5
}

Response:

  • HTTP status code: 200
{
  "representation": "hsv",
  "color": [
    128,
    85,
    95
  ],
  "operation": "desaturate",
  "modified_color": [
    128,
    81,
    95
  ]
}
