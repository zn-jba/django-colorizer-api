Description

Your API can handle saturation and desaturation but there is always a window for new features that can complement your API!
Add a view that handles color conversion between HSV and RGB. Rapidtables provides concise formulas that can be transferred to Python. You can also use their calculator to check whether your code works correctly:

  • RGB to HSV
  • HSV to RGB

Until now, your API has been unable to validate color values and would accept any value as is. For example, an HSV color with parameters [700, 150, 200] is not valid since hue (H) is above 360, saturation (S) is above 100, and value (V) is above 100. You need to check the data in the request as well as the data in the response to make sure that the color values are correct. Use convert-color/ path_info for URL mapping.

Correct values:

HSV:

\(0 \leq hue \leq 360\newline 0 \leq saturation \leq 100\newline 0 \leq value \leq 100\newline\)

RGB:

\(0 \leq red \leq 255 \newline 0 \leq green \leq 255 \newline 0 \leq blue \leq 255 \newline\)

Objectives

  1. Create a view that receives an RGB or an HSV color and a target representation, HSV or RGB respectively.
  2. Validate the color data and make sure the data you send is valid.

Examples

The greater-than symbol followed by a space (> ) represents the user input. Note that it's not part of the input.

Example 1: A correct request.

Request:

{
  "representation": "hsv",
  "color": [
    128,
    85,
    95
  ],
  "conversion": "rgb"
}

Response:

  • HTTP status code: 200
{
  "color": [
    128.0,
    85.0,
    95.0
  ],
  "converted_color": [
    36,
    242,
    64
  ]
}

Example 2: A wrong request.

Request:

{
  "representation": "hsv",
  "color": [
    190,
    150,
    150
  ],
  "operation": "saturate",
  "amount": 5
}

Response:

  • HTTP status code: 400
{
    "error": "Invalid data."
}
